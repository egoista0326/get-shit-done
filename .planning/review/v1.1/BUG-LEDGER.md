# v1.1 Bug Ledger

**Milestone:** v1.1 Skill Verification
**Purpose:** User-facing record of every confirmed bug fixed during the review milestone.
**Policy:** Candidate findings are not recorded as fixed bugs until a second-pass confirmation accepts them. Each accepted bug must include root cause, affected files, fix summary, verification, and review round.

## Status

Round 1 confirmed and fixed four bugs. Phase 19 then passed Round 2 and Round 3 consecutively clean. The user then requested one extra stricter review pass with richer scenarios and prompt-quality scrutiny. Phase 20 / Round 04 confirmed and fixed additional bugs, which reset the clean counter. Round 05 found additional confirmed issues in evidence type checks, chained research-pipeline insertion, and status documentation. Round 06 then found further issues in lifecycle gates, migration backup/release safety, research-pipeline reuse, experiment-integrity semantics, and generated prompt capability preservation. Round 07 found additional strict-review issues, including Claude Code -> Codex conversion drift and official Codex-source capability gaps. Round 08 then found additional lifecycle/quality-gate false-clean risks, post-fix rerun policy drift, prompt-fidelity drift, and a result-to-claim staleness interpretation that was later superseded. Round 09 found additional Codex interface, state/path safety, migration proof, prompt-fidelity, and experiment evidence issues. Round 10 found additional prompt-capability, evidence-alias, code-review-fix, pause/resume, path-safety, and migration-symlink issues. Round 11 found final accounting, evidence-alias, dangling-symlink, and migration report-target issues. All confirmed Round 10/Round 11 issues have been fixed, but the bounded loop reached the 11-round cap without two consecutive clean rounds, so the final strict-review success criterion is not met.
