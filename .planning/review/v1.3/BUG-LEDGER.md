# v1.3 Bug Ledger

**Purpose:** record confirmed issues found during the two-stage post-v1.2 review.

## Policy

- P0/P1/P2 issues reset the active stage clean count to 0 after they are fixed.
- P3/minor issues may be fixed while still counting a round as clean only if they are extreme, rare, or normal-use-non-impacting.
- Every accepted issue must include second-pass confirmation against upstream GSD, upstream Auto/ARIS, accepted `ljx-GSD` design, generated preview/install contract, or runtime correctness.
- Every accepted issue must include a fix summary, affected files, verification evidence, and status.
