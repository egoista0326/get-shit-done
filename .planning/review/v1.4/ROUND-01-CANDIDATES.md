# v1.4 Stage A Round 1 Candidates

**Round scope:** independent runtime closure, generated skill self-containment, prompt fidelity, artifact traceability, config/variable propagation, Codex hook conformance, lifecycle safety, and Stage B scenario readiness.

**Review sources:** local preview install, local scans, official Codex hook/skill docs, and six focused subagent reviews.

## Confirmed For Fix

| Candidate | Severity | Confirmation | Planned handling |
| --- | --- | --- | --- |
| Raw upstream Auto/ARIS skills are installed as active top-level callable skills, and some raw skills call other raw Auto skills. | P1 | Local preview showed non-`ljx-GSD-*` `skills/*/SKILL.md` entries; Curie confirmed raw call chains such as `idea-discovery -> research-lit` and `paper-writing -> paper-*`. Official Codex skills docs say any directory with `SKILL.md` is an invocable skill surface. This violates the new v1.4 hard invariant. | Remove active top-level upstream Auto skill installation; keep upstream material only under `ljx-gsd/upstream-auto/` reference/internal assets and prune stale ljx-managed raw skills. |
| Upstream Auto support assets are installed under active top-level roots (`tools/`, `templates/`, `mcp-servers/`, `skills/shared-references`) instead of the `ljx-gsd/` internal tree. | P1 | Curie confirmed manifest paths and build target roots. The v1.4 protocol allows bundled upstream-derived assets only through the `ljx-gsd/` tree. Generated `ljx-GSD-*` skills do not reference those top-level roots, so they can be internalized. | Move upstream support assets into `ljx-gsd/upstream-auto/`; keep only generated `ljx-GSD-*` skills as active skill surface. |
| Existing unmanaged raw Auto skills poison ljx-GSD install by triggering same-name conflict checks. | P1 | Curie confirmed `assertNoUnmanagedAutoSkillConflicts` and tests expect install abort when `skills/idea-discovery` already exists. New user requirement says ljx-GSD must coexist with installed GSD/Auto but remain inside its own system. | Stop treating raw Auto names as install targets. Preserve unmanaged raw skills, remove only prior ljx-managed raw skills. |
| Codex hook template uses TOML `[[hooks]]`, unsupported events, and `CODEX_PROJECT_DIR` instead of current Codex `hooks.json` plus stdin `cwd`. | P1 | Mendel compared generated preview with OpenAI Codex hook docs. Official docs require `hooks.json`, supported events such as `SessionStart`, `PreToolUse`, `PostToolUse`, `UserPromptSubmit`, `Stop`, and JSON input with `cwd`. | Generate a Codex `hooks.json` reference template and update meta-opt scripts to resolve project cwd from hook stdin JSON. |
| `research_stage_handoff` is persisted but not merged into effective runtime config. | P1 | Mendel fixture found handoff values present in phase records but ignored by `loadProjectConfig`, losing pipeline-created `autoProceed`, checkpoint, compact, review, paper, and compute policies. | Normalize `research_stage_handoff` into runtime config between phase overrides and explicit CLI overrides. |
| Documented direct helper CLI flags are silently ignored. | P2 | Mendel showed review-loop, experiment-bridge, and paper-pipeline parsers accept only `--cwd`/`--phase` while parameter docs mark workflow flags as CLI-compatible. | Add shared CLI override parsing or fail loudly on unsupported documented flags; prefer applying overrides for supported research flags. |
| Paper pipeline ignores shared defaults/legacy continuation aliases when paper-specific keys are unset. | P2 | Mendel fixture showed `AUTO_PROCEED=true` / `HUMAN_CHECKPOINT=false` and paper phase workflow overrides still emitted safe defaults. | Add paper fallback to `research.defaults.*` and map paper-phase workflow overrides to paper-specific keys. |
| Autonomous profile changes code-review rerun policy to `automatic` despite the locked `recommend` contract. | P2 | Mendel found design/config lock `recommend`, while profile preset and tests expect `automatic`. | Keep `recommend` unless explicit config opts into `automatic`. |
| Lifecycle phase-record route overrides can emit raw non-`ljx-GSD-*` commands. | P2 | Curie confirmed `primary_command`, `plan_command`, `execute_command`, and `advanced_direct_tools` are copied without ljx-only validation. | Reject or ignore raw override command names and surface deferred invalid overrides without recommending raw skills. |
| Migration helper actions use source-tree `node bin/lib/...` paths and legacy Auto import is not reachable from documented CLI surface. | P2 | Kant confirmed installed runtime helper exists but safe actions point to source checkout; Meitner confirmed CLI lacks source import flags while library tests call direct APIs. | Use installed/current helper path in safe actions and add CLI source flags/import command for operator-visible migration. |
| Auto research artifacts are classified as generic residue or omitted from structured ledgers. | P1/P2 | Boole confirmed `FINAL_PROPOSAL`, `EXPERIMENT_LOG`, `AUTO_REVIEW`, `REVIEW_STATE`, `paper/main.tex`, `paper/main.pdf`, and related files are not typed consistently. | Expand migration classifier and workflow evidence links so Auto artifacts remain traceable without becoming the control plane. |
| Experiment audit prompt names the audit stage but drops concrete Auto `experiment-audit` checks. | P2 | James compared generated `ljx-GSD-experiment-bridge` with upstream `experiment-audit`; missing fake-ground-truth, self-normalized metric, phantom result, scope inflation, and independent reviewer checks. | Strengthen experiment-bridge prompt floor and tests. |
| Blocked workstream mutations compute migration helper context but drop it from the returned stop payload. | P3 | Kant confirmed `buildMutationStop` ignores the passed `migrationStatus`. | Preserve migration status/helper action payload in blocked mutation output. |
