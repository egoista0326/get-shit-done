# v1.4 Bug Ledger

**Purpose:** Record confirmed issues found during the self-contained runtime and scenario review.

## Policy

- P0/P1/P2 findings reset the active stage clean count to 0 after fixes.
- P3/minor findings may be fixed while preserving clean status only when they are extreme, rare, or normal-use-non-impacting.
- Every accepted issue must include second-pass confirmation.
- Every accepted issue must include root cause, fix summary, affected files, verification, and status.
- `.planning` accounting-only changes do not reset clean count unless they affect skill behavior, runtime/install routing, artifact lookup, state recovery, config inheritance, or verification reliability.
