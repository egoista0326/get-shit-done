<threat_model>
Trust boundaries: This plan can define implementation policy and candidate reuse classes, but it must not silently approve risky reuse or alter final review gates.

Threat register:
- T-04-02-01: Implementation begins from fuzzy ownership boundaries. Mitigation: require explicit ownership and mutation rules by surface.
- T-04-02-02: ljx-GSD reuse is assumed safe because it seems convenient. Mitigation: classify every current-code candidate as candidate-only pending Phase 05.
- T-04-02-03: Repo-copy strategy conflicts with review discipline or upstream upgradeability. Mitigation: boundary rules must include git, hook, naming, and review-command behavior.
- T-04-02-04: Boundary docs duplicate framework docs without adding implementation clarity. Mitigation: boundary outputs must focus on ownership, mutation rules, and workspace strategy.
</threat_model>
