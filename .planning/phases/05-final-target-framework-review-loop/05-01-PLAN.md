<threat_model>
Trust boundaries: 05-01 prepares review infrastructure only. It must not start implementation, silently relax stop gates, or allow review accounting to drift during Phase 05.

Threat register:
- T-05-01-01: Review starts before parser/accounting and lane scope are frozen. Mitigation: define review artifact layout, round lifecycle, and clean-round/reset rules explicitly in the harness.
- T-05-01-02: Review lanes ignore Phase 04 evidence and re-review the framework abstractly. Mitigation: require Phase 04 feasibility, boundaries, reuse, and workspace docs as mandatory probes.
- T-05-01-03: Review loop mutates implementation surfaces instead of framework docs. Mitigation: harness must define allowed fix surfaces and forbid code implementation in Phase 05.
- T-05-01-04: Findings are written in inconsistent formats, recreating parser/accounting drift. Mitigation: prompts must require one stable finding schema and deterministic status fields.
</threat_model>
