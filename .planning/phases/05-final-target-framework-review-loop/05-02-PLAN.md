Trust boundaries: 05-02 may update framework/evidence docs in response to accepted findings, but it must not change implementation code, review accounting rules, or the basic GSD-first architecture without explicit accepted findings.

Threat register:
- T-05-02-01: Review lanes produce inconsistent finding formats or incomplete coverage. Mitigation: use frozen lane prompts and round-state accounting before counting a round.
- T-05-02-02: Main-agent synthesis accepts weak or duplicate findings. Mitigation: accepted findings require explicit second-pass confirmation with rule, evidence, and required change.
- T-05-02-03: Round fixes drift into implementation or hidden reuse. Mitigation: restrict changes to Phase 02 framework docs and only the specific Phase 04 evidence docs implicated by findings.
- T-05-02-04: Clean streak is counted dishonestly after findings or rule changes. Mitigation: reset on accepted P0/P1/P2 findings and on material harness/parser changes; cap at 15 rounds.
</threat_model>
