# 01-01 Plan: Run Subagent Build Round And Synthesize Initial Source Frameworks

## Objective

Run exactly one source-framework build round through isolated subagents, then synthesize initial Phase 01 framework artifacts from subagent reports only.

## Scope

Subagents read upstream GSD, current ljx-GSD/history, and Auto/ARIS source systems. Main agent coordinates, collects reports, and writes phase-local framework artifacts. Main agent does not replace missing source-reading lanes.

## Build Lanes

- Upstream GSD lifecycle/control-plane: workflows, skills, milestones, phases, `next`, progress, verification, autonomous behavior.
- Upstream GSD runtime/helper substrate: package/install/update, runtime scripts, config, state, hooks, git, tests, subagents, templates.
- Current ljx-GSD implementation: build/install/generation, runtime helpers, state/config/lifecycle/quality/research helpers, tests.
- Current ljx-GSD history: v1.4 pivot, archived phases, phase records, v1.1-v1.4 review artifacts, failure taxonomy.
- Auto/ARIS front half: literature, idea discovery, novelty, review, refinement, experiment planning, source helpers, front-half artifacts.
- Auto/ARIS back half: experiment bridge/audit, execution evidence, claims, ablations, paper/rebuttal, result analysis.
- Auto/ARIS paper/rebuttal/tooling/Codex variants: run in a later batch if subagent slots are exhausted.

## Execution Steps

1. Collect all currently running build-lane subagent reports.
2. If any build lane is missing because the subagent limit was reached, wait for a slot and spawn a later-batch subagent for that lane.
3. Record which lanes completed, which files they inspected, and which source surfaces they skipped.
4. Synthesize initial framework artifacts from subagent reports only.
5. Mark any unsupported source claim as a gap, not as extracted fact.
6. Do not write a completion summary until review rounds 1-3 and the main-agent final check are complete.

## Acceptance Criteria

- Every required source system has at least one completed subagent lane.
- Missing lanes are explicitly run in later subagent batches; local main-agent scans are not counted as evidence.
- Initial framework docs cite source paths and distinguish facts from recommendations.
- Context hygiene log records subagent limits, skipped files, and pending follow-up checks.
- No implementation edits are made.
