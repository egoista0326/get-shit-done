<threat_model>
Trust boundaries: final synthesis is a serialized main-agent write. It may define future implementation contracts but must not edit implementation code or canonical project state until the user accepts the framework.

Threat register:
- T-02-03-01: Main synthesis invents consensus that proposal rounds did not support. Mitigation: cite the proposal or context source for each major framework decision and mark unsupported items as deferred.
- T-02-03-02: The target framework becomes a second workflow system. Mitigation: include a no-phase-type/no-second-control-plane compatibility note with explicit rejected alternatives.
- T-02-03-03: Completion semantics remain vague and allow future false completion. Mitigation: define raw evidence, review/verify gates, advisory signals, and danger-auto override statuses separately.
- T-02-03-04: danger-auto grants broad operational capability without auditability. Mitigation: require run log, authorization actions, override log, side effects log, missing authorization behavior, and provisional status propagation.
- T-02-03-05: Framework acceptance happens without user review. Mitigation: include a blocking human-review checkpoint before writing the plan summary or marking Phase 02 complete.
</threat_model>
