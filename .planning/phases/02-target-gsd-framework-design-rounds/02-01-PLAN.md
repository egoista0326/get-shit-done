<threat_model>
Trust boundaries: proposal subagents may read broad source material but may write only their assigned Phase 02 proposal file. Canonical project files remain owned by the main agent.

Threat register:
- T-02-01-01: A proposal subagent mutates canonical state, roadmap, requirements, or Phase 02 context. Mitigation: assign disjoint proposal file ownership and reject any changes outside files_modified.
- T-02-01-02: A proposal invents source facts or treats user decisions as optional. Mitigation: require source references to 02-CONTEXT.md and Phase 01 artifacts, and mark inference separately from extracted fact.
- T-02-01-03: The first round silently reintroduces phase_type, typed routing, or a second control plane. Mitigation: require every proposal to state whether it changes phase schema and why it does not.
- T-02-01-04: danger-auto gets under-specified and later becomes unsafe or dishonest. Mitigation: require explicit side-effect, authorization, override, and degraded-completion semantics in at least one proposal lane.
</threat_model>
