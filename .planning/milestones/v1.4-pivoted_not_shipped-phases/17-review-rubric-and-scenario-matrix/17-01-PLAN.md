# 17-01: Define Review Dimensions And Blocking-Severity Policy

<objective>
Create a severity-ordered rubric that reviewers can apply consistently before fixes begin.
</objective>

## Tasks

- Define review dimensions from hard failures through polish.
- Define P0/P1/P2/P3 severity meanings and pass/fail gates.
- Tie each dimension to evidence sources and bug triggers.
- Write `.planning/review/v1.1/REVIEW-RUBRIC.md`.

## Verification

- The rubric starts with hard bugs and data-loss risks and ends with low-risk polish.
- The rubric includes parity, state ownership, generated-output, minimal-modification, and efficiency checks.
