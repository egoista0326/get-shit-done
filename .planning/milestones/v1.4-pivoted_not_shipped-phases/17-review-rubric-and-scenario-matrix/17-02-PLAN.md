# 17-02: Define Scenario Probes And Evidence Matrix

<objective>
Turn the user's requested common scenarios into a concrete review matrix.
</objective>

## Tasks

- Enumerate the user-requested scenarios and map each to relevant files/helpers/tests.
- Define lightweight probes that do not require expensive external execution by default.
- Define pass criteria and bug triggers for each scenario.
- Write `.planning/review/v1.1/SCENARIO-MATRIX.md`.

## Verification

- The matrix covers GSD migration, engineering flow, research flow, code review, literature review, automatic research loop, result analysis, idea generation, pause/resume, workstreams, workspaces, policy switching, and parameters.
- Every scenario names evidence requirements and what counts as failure.
