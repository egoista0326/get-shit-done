# 17-03: Define Review Loop, Confirmation, And Bug-Ledger Protocol

<objective>
Define the bounded review/fix loop so Phase 18 and Phase 19 can run autonomously without drifting standards.
</objective>

## Tasks

- Define candidate finding intake, second-pass confirmation, rejected-candidate logging, and accepted-fix recording.
- Define clean-round accounting and the 11-round cap.
- Define how bug ledger entries must be written for the user.
- Write `.planning/review/v1.1/REVIEW-PROTOCOL.md`.

## Verification

- The protocol requires second-pass confirmation before modifications.
- The protocol resets clean streak on any confirmed bug and stops only after two clean rounds or Round 11.
- The bug ledger fields match the user's requested description/root-cause/fix-summary style.
