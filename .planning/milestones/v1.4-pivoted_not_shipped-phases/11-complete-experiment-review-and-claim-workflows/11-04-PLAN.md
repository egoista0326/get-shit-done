<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Result artifacts to claim judgment | Experiment/review artifact presence influences claim support and readiness summaries. |
| Internal evaluator concepts to claim-gate output | `claim-gate` names evaluators but must not mark them complete without artifacts. |
| Runtime semantics to generated skills | Generated skill text can mislead users if it describes stale bridge-era behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04-01 | Spoofing | `claimReadiness.evaluatorArtifacts` | mitigate | Derive evaluator status only from exact phase-local `RESULT_TO_CLAIM` and `RESEARCH_REVIEW` artifact links, and mark stale `RESULT_TO_CLAIM` when newer results or critique artifacts exist. |
| T-11-04-02 | Tampering | Claim state context | mitigate | Expose `claimStateRecordPath` through the shared helper, do not write claim JSON during context reads, and restrict writes to explicit support-judgment/readiness writer functions with distinct `updatedBy` fields. |
| T-11-04-03 | Information Disclosure | Generated skill/install output | mitigate | Extend `tests/skill-build.test.cjs` and `node bin/install.js --preview` to keep wording and installed runtime aligned without leaking source paths. |
| T-11-04-04 | Repudiation | Claim-gate recommendations | mitigate | Generated wording and tests must state claim-gate does not silently run or complete `result-to-claim` or `research-review`. |
| T-11-04-05 | Elevation of Privilege | Downstream paper routing | mitigate | Keep `paper-pipeline`, `ablation-planner`, and `experiment-plan` as explicit recommendations only per D-21. |
</threat_model>
