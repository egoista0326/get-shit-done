<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI phase input to state path | Phase IDs and project roots can influence local `.planning/state` paths. |
| Phase-local artifact scan to structured summary | File presence is summarized into state-link metadata used by later routing. |
| Repo runtime to installed runtime | `build-skills` copies helper code into a generated install tree. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01-01 | Tampering | `buildStateRecordPath()` family use | mitigate | Add only centrally allowlisted `experiments` and `claims`; tests assert unsupported families still throw. |
| T-11-01-02 | Spoofing | Evidence artifact links | mitigate | Build links only from resolved `baseContext.phase.path` and `baseContext.phase.id`, not user-supplied arbitrary paths. |
| T-11-01-03 | Repudiation | Evidence summary freshness | mitigate | Helper returns link/status/freshness summaries only and does not write empty state during context reads; stale `RESULT_TO_CLAIM` is derived from `mtimeMs` comparisons and later writers must set explicit `updatedBy` fields. |
| T-11-01-04 | Information Disclosure | Installed runtime build output | mitigate | Extend `tests/skill-build.test.cjs` and preview install checks so helper copying does not bypass existing absolute-path leak protections. |
</threat_model>
