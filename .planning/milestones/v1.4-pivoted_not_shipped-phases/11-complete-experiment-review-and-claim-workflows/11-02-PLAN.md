## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI phase input to experiment helper context | User-provided phase arguments select phase directories and artifact paths. |
| Experiment plan artifact to execution bridge | A phase-local markdown artifact gates bridge execution context. |
| Evidence link summary to downstream routing | Present/missing links can influence later review and claim recommendations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-02-01 | Tampering | `readExperimentBridgeContext()` | mitigate | Preserve `missing_experiment_plan` stop before bridge context proceeds; never fabricate `EXPERIMENT_PLAN`. |
| T-11-02-02 | Repudiation | Evidence state paths | mitigate | Context exposes intended paths but does not create state records, preventing false evidence provenance. |
| T-11-02-03 | Spoofing | Phase artifact paths | mitigate | Use `readPhaseWorkflowContext()` and shared evidence helper from resolved `baseContext.phase`, not user-provided raw paths. |
| T-11-02-04 | Denial of Service | Companion execution routes | accept | This plan does not launch experiments; execution remains explicit through companion skills. |
</threat_model>
