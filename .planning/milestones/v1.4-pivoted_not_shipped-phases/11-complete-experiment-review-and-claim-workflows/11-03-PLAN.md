| Boundary | Description |
|----------|-------------|
| Review artifact presence to evidence links | File presence is summarized into downstream evidence metadata. |
| Review-loop state writer to runtime state | Iterative review state is persisted under the `reviews` family. |
| Single-round review context to downstream claims | A one-shot critique can influence claim routing but must not spoof a full review loop. |

| T-11-03-02 | Tampering | `writeReviewLoopState()` | mitigate | Keep writes in `reviews` only and assert no `experiments`/`claims` records are created by the review writer. |

| T-11-03-04 | Elevation of Privilege | Review helper routing | accept | Helpers expose recommendations only; they do not run paper, claim, or experiment companion commands in this plan. |
</threat_model>
