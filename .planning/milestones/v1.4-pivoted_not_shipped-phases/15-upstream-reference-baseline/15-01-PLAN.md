# 15-01: Establish Upstream Repository Snapshots And Source Inventory

<objective>
Record the upstream repository snapshots and create an inventory of the source areas that later Phase 15 notes depend on.
</objective>

## Tasks

- Confirm both upstream repositories are cloned under `.planning/references/upstreams/`.
- Record each repository's commit head.
- Inventory behavior-defining files for GSD and Auto/ARIS, prioritizing workflows, skills, commands, agents, runtime helpers, shared references, templates, tools, and tests.
- Write the resulting inventory to `.planning/review/v1.1/UPSTREAM-SOURCE-INVENTORY.md`.

## Verification

- The inventory file contains clone paths, commit heads, source categories, and the inspected-file shortlist.
- Later Phase 15 plans can cite the inventory rather than repeating discovery.
