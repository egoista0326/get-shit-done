## Issues Driving The Pivot

The user reported or confirmed these failure families:

- Planning produced a system that looked GSD-like but did not reliably preserve GSD lifecycle semantics.
- `ljx-idea-discovery-tools` exposed state/context writing but did not itself perform full evidence-producing literature execution.
- `autoProceed=true` and stop-boundary semantics did not produce expected automatic progression.
- Engineering phases could still stop in confusing intermediate states, e.g. treating plan-inventory routing as direct refine.
- `next` could indicate bridge readiness without verifying phase-goal completion.
- Phase records, roadmap checkboxes, `STATE.md`, and summary artifacts drifted apart.
- Research-related flows did not consistently use the same git, hook, lifecycle, artifact, and state discipline as GSD workflows.
- Concurrent lifecycle synchronization caused write races and status overwrites.
- Some phase artifacts were written without the real execute evidence chain required by the user.
- Existing review rounds found repeated defects across generated skills, runtime helpers, config propagation, prompt fidelity, self-containment, safety gates, and state mirrors.

## Carry-Forward Constraints

The next milestone should preserve these constraints:

- Use `gsd` as the external and internal name. The user's later `egoista` wording is a placeholder for the new in-house GSD, not a naming decision.
- Start from upstream GSD behavior and implementation patterns, not from a greenfield rewrite.
- Integrate Auto/ARIS research as standalone `gsd` commands backed by ordinary GSD phases and a narrow research artifact convention, not as a second top-level control plane.
- Do not add `phase_type`, typed phase routing, or broad phase schema changes; the previous typed-phase approach had too much blast radius.
- Treat upstream GSD, current ljx-GSD, archived milestone docs, and Auto/ARIS as source material that must be framework-indexed before implementation.
- Review rules must be derived from real historical bugs and must be discussed before the framework review loop starts.
- No completed state may be written without an evidence-producing execute path and verification gate.
